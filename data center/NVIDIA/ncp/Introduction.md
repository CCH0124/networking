# Introduction


本文件概述了一份軟體參考指南，旨在協助 NVIDIA 雲端合作夥伴 (NCP)、雲端服務供應商 (CSP) 和獨立軟體供應商 (ISV) 在 NCP 硬體平台上建置 AI 雲端服務。它提供了一個支援多租戶與彈性資源分配的基礎設施原生 (Infrastructure-native) 北極星 (Northstar) 參考架構。

隨著推論工作負載的重要性日益增加，企業組織對於能夠動態滿足多租戶訓練與推論工作負載的通用運算資源需求也與日俱增。這一轉變推動了對建置和運行 AI 服務堆疊的雲端原生方法之需求。

本文件描述了如何使用 Kubernetes 和現代 AI 平台即服務 (PaaS) 方案來建置此類解決方案。本文件還提供了在虛擬化環境中最佳化 AI 推論和訓練效能的指引，其中工作負載可以在共享或專屬的實體主機上運行。

該架構堆疊建置於 [NCP 硬體參考設計 (NCP Hardware Reference Design)](https://www.nvidia.com/en-us/data-center/gpu-cloud-computing/partners/) 之上，採用分層方式：

* **基礎設施即服務 (IaaS)**：用於裸機 (Bare metal) 與虛擬機 (VM) 的佈署
* **容器即服務 (CaaS)**：搭配託管式 Kubernetes
* **AI 平台即服務 (PaaS)**：用於面向租戶的 AI 工作負載

分層設計使 NCP 能夠提供具備與超大規模雲端服務供應商競爭力的動態、多租戶 AI 服務。該架構是基礎設施原生的；運算、儲存和網路資源是以隨需 (On-demand) 模式進行分配，而非靜態配置。

NCP 可以與整合服務供應商 (ISV) 生態系統合作，或導入開源工具來提供安全性和工作負載隔離。這使得運行並行的多租戶私有雲成為可能，並能充分利用本參考架構中概述的效能最佳化堆疊。NVIDIA 對第三方解決方案的效能進行認證，以便 NCP 可以信心十足地選擇他們首選的合作夥伴。

## 文件結構 (Document Structure)

本文件組織為兩個章節：

* [第一部分：軟體參考指南 (Part 1: Software Reference Guide)](/dsx/ncp/part-1-software-reference-guide/ncp-software-reference-guide)：描述了分層的多租戶 AI 雲端架構，涵蓋了每個層級（IaaS、CaaS 和 PaaS）所需的能力。其中提及了對於網路管理、工作負載隔離、可觀測性 (Observability) 以及故障修復 (Break-fix) 的考量。
* [第二部分：軟體元件 (Part 2: Software Components)](/dsx/ncp/part-2-software-components/nvidia-software-components)：提供有關 NVIDIA 提供的軟體之詳細資訊，以解決「軟體參考架構」章節中提及的特定能力。每個項目皆解釋了該軟體元件，並將其對應回架構能力。使用 NVIDIA 提供的軟體是選配的，取決於架構決策。

## 對象與範圍 (Audience and Scope)

本文件是一份參考指南，而非實作手冊。它描述了各層所需的能力，並指出可以整合 NVIDIA 提供的軟體之處。

* **NCP**：將此作為營運多租戶 AI 雲端服務所需能力的檢核清單。每個章節描述了各層所需的關鍵能力。此參考架構可用於評估您目前的軟體供應堆疊，以識別差距並做出明智的建置或購買決策。
* **ISV**：將您的軟體產品對應至此分層架構框架。每個章節中指出的 NVIDIA 軟體代表了您所提供解決方案的整合點和互補元件。

## 使用者角色 (User Personas)

下表列出的使用者角色是與使用 NCP 軟體參考指南實作的系統進行互動、管理或使用的角色。

**使用者角色**

| 角色 (Persona) | 定義 (Definition) |
| -------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 供應商 (Vendor) | 這是 NVIDIA 或整合到 NCP 產品中的軟體、硬體或服務的第三方供應商。 |
| 營運商 (Operator) | 在 NCP 部署中擁有並營運（實體與虛擬）基礎設施的實體。該實體負責 IaaS/CaaS/PaaS 平台服務、租戶隔離、安全性以及故障修復操作。 |
| 租戶 (Tenant) | 在叢集中消耗資源的實體，代表 NCP 的客戶。租戶向 NCP 購買服務，並直接使用 NCP 提供的服務，或在其之上建置服務。租戶包含兩類使用者：租戶管理員和租戶終端使用者。 |
| 管理員 (Administrators) | 租戶管理員負責在 NCP 平台的基礎上管理租戶的服務。 |
| 終端使用者 (End User) | 租戶終端使用者代表租戶的客戶。終端使用者也可以是租戶組織內的使用者（例如：員工）。此群體包括開發和佈署模型的 AI 從業者（機器學習工程師、資料科學家），或消耗 AI 驅動服務的應用程式使用者。 |

## 術語與定義 (Terms and Definitions)

本文件中的術語和定義如下表所示。

**術語與定義**

| 術語 (Term) | 定義 (Definition) |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| NVIDIA 雲端合作夥伴 (NCP) | 利用 NVIDIA 平台建置資料中心並提供 AI 雲端服務的 NVIDIA 合作夥伴。 |
| Kubernetes 即服務 (KaaS) | 一種為終端客戶快速建立 K8s 叢集的自動化機制。 |
| 並行多租戶 (Concurrent multitenancy) | 在任何時間點，有多個租戶（直接或間接）在同一個實體主機上處於活動狀態或擁有存取權限的場景。 |
| 工作主機 (Worker host) | 構成控制平面以及工作主機基礎的裸機伺服器。 |
| 工作節點 (Worker node) | 實體主機上的虛擬化運算資源，通常稱為「客端虛擬機 (Guest Virtual Machine)」。工作節點是租戶用來執行其工作負載的資源。 |
| 共享主機 (Shared host) | 兩個租戶共享的工作主機（主機上的並行多租戶）。 |
| 專屬控制平面 (Dedicated control plane) | 在叢集內僅運行單一租戶工作負載的 K8s 控制平面（或類似機制）。 |
