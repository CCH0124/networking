# Ethernet and TCP/IP 與 InfiniBand

在架構設計上，將 Ethernet 比喻為「高速公路系統」，它能容納各式各樣的車輛（通用流量），但容易受到交通號誌與壅塞的影響；而 InfiniBand 則如同「專屬的高鐵軌道」，停靠站少且專為極速傳輸設計，但僅限於特定路線與應用場景。

以下為兩者在核心架構與效能指標上的詳細比較表格：

| 比較維度 (Feature) | Ethernet & TCP/IP 堆疊 | InfiniBand 架構 |
| :--- | :--- | :--- |
| **設計初衷與應用場景** | 通用網路（LAN, WAN, 網際網路、Web 應用） | 高效能運算 (HPC)、AI/ML 叢集、資料中心內部互連 |
| **傳輸延遲 (Latency)** | 較高（典型值約 10 ~ 100 微秒） | 極低（約 1 ~ 2 微秒） |
| **通訊協定與卸載** | 依賴作業系統的 TCP/IP 堆疊，會產生較高負擔 | 依賴 RDMA (遠端直接記憶體存取)，繞過 CPU，負擔極低 |
| **資料搬移機制** | 基於記憶體複製 (Copy-based) | 零複製 (Zero-copy)，硬體直接讀寫記憶體 |
| **CPU 運算負擔** | 高（需處理核心切換與大量中斷） | 極低（硬體層級處理 RDMA 卸載） |
| **流量控制 (Flow Control)** | 基於滑動視窗 (Window-based)，允許掉包後重傳 | 基於信用額度 (Credit-based)，設計為絕對無損 (Lossless) 網路 |
| **路由機制 (Routing)** | 分散式計算（如 OSPF, BGP 等 IP 路由協定） | 集中式控制（由 Subnet Manager 統一計算與下發路由） |
| **可靠性保證** | 軟體層級保證（TCP 封包重傳機制） | 硬體層級保證（網路卡晶片內的 Reliable Connection, RC） |
| **連線處理方式** | 基於 Socket | 基於 Queue Pairs (QP) |
| **擴展性與彈性** | 網際網路規模，隨插即用，通用相容性極高 | 資料中心與叢集規模，需要專屬的驅動與管理軟體 |
